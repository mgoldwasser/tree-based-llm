\begin{abstract}
We investigate replacing the dense linear projections (Q, K, V) in transformer attention with differentiable soft decision trees, trained end-to-end via backpropagation. We introduce several tree-based projection architectures---\textit{BatchedTreeForest}, \textit{ObliviousTreeForest} (NODE-style), and \textit{LinearPlusForest} (linear base + tree correction)---all computed via batched einsum operations. Through systematic ablation on character-level Shakespeare language modeling, we find that the Linear+Forest architecture with unfused QKV routing matches a standard transformer at \textbf{38.3\% vs.\ 38.7\%} validation accuracy, while speed-optimized variants using oblivious trees, alternating layers, and selective V+O tree placement achieve comparable accuracy at significantly reduced cost (\textbf{171ms vs.\ 545ms} per step). We also evaluate a Mixture-of-Experts variant replacing tree routing with top-k expert selection. Our analysis reveals that soft routing---not hard tree decisions---is the key feature: tree forests compute input-adaptive linear projections where each input sees a different effective weight matrix.
\end{abstract}

% ══════════════════════════════════════════════════════════════════════════
\section{Introduction}

The transformer architecture \citep{vaswani2017attention} relies fundamentally on linear projections to compute queries, keys, and values for attention. These projections are simple matrix multiplications---computationally efficient but limited to learning linear relationships between input features.

Decision trees offer a compelling alternative: they learn piecewise-constant functions through hierarchical feature partitioning, can capture non-linear relationships naturally, and provide interpretable routing decisions. However, classical decision trees use hard (non-differentiable) splits, making them incompatible with gradient-based training.

\textit{Soft decision trees} \citep{irsoy2012soft,frosst2017distilling} resolve this by replacing hard splits with sigmoid-gated routing, allowing gradients to flow through all tree paths simultaneously. We build on this foundation to create tree-based projection layers that serve as drop-in replacements for linear layers in transformer attention.

Our key contributions:
\begin{enumerate}
    \item \textbf{BatchedTreeForest and ObliviousTreeForest}: Batched implementations of standard and NODE-style oblivious decision trees, computing all trees via single einsum operations.
    \item \textbf{LinearPlusForest}: A linear base projection augmented with a single wide tree forest for nonlinear correction.
    \item \textbf{Speed optimization ablation}: Oblivious trees, alternating layers, selective V+O placement, and \texttt{torch.compile} reduce the speed gap while preserving accuracy.
    \item \textbf{MoE comparison}: A Mixture-of-Experts variant replacing tree routing with top-k expert gating.
    \item \textbf{Empirical validation} on Shakespeare across 6 model configurations at full scale ($d_{\text{model}}=128$, 4 layers, 256 sequence length).
\end{enumerate}

\section{Background}

\subsection{Soft Decision Trees}

A soft decision tree of depth $D$ has $2^D - 1$ internal nodes and $2^D$ leaves. Each internal node $i$ computes a soft routing probability:
\begin{equation}
    p_i^{\text{left}}(\mathbf{x}) = \sigma\!\left(\frac{\mathbf{w}_i^\top \mathbf{x} + b_i}{\tau}\right)
\end{equation}
The probability of reaching leaf $l$ is the product of routing decisions along the path:
\begin{equation}
    P(l \mid \mathbf{x}) = \prod_{i \in \text{path}(l)} p_i^{d_i}(\mathbf{x})
\end{equation}
The tree output is a probability-weighted sum over learned leaf vectors:
\begin{equation}
    f(\mathbf{x}) = \sum_{l=1}^{2^D} P(l \mid \mathbf{x}) \cdot \mathbf{v}_l
\end{equation}
Crucially, with soft routing, the forest computes an \textit{input-adaptive linear projection}---a different effective weight matrix for every input, constructed as a weighted combination of leaf matrices.

\subsection{Oblivious Decision Trees (NODE-style)}

In oblivious trees \citep{popov2020neural}, all nodes at the same depth share one hyperplane. Decision weights have shape $(T, D, D_{\text{in}})$---fewer parameters than standard trees' $(T, 2^D-1, D_{\text{in}})$. Leaf probabilities are computed via outer product of per-depth choices, eliminating depth-dependent gradient vanishing.

\subsection{Transformer Attention}

Standard multi-head attention computes $Q = \mathbf{x} W_Q$, $K = \mathbf{x} W_K$, $V = \mathbf{x} W_V$, then:
\begin{equation}
    \text{Attn}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V
\end{equation}
We replace the linear projections $W_Q, W_K, W_V$ with tree-based projection layers.

\section{Method}

\subsection{BatchedTreeForest}

All tree parameters are stored in stacked tensors. The routing computation for all $T$ trees is a single batched einsum:
\begin{equation}
    \text{decisions} = \texttt{einsum}(\texttt{`bsd,tnd->bstn'}, \mathbf{x}, W_{\text{decision}})
\end{equation}
Leaf probabilities are computed in log-space. Input-dependent gating via a learned projection replaces fixed mixture weights, allowing each token to route to different tree combinations (MoE-style).

\subsection{ObliviousTreeForest}

Oblivious trees constrain all nodes at the same depth to share a single hyperplane, reducing decision weights from $(T, 2^D\!-\!1, D_{\text{in}})$ to $(T, D, D_{\text{in}})$. Leaf probabilities are computed via Kronecker product, unrolled for depth~3:
\begin{equation}
    P(\text{leaf}_{ijk}) = d_0^i \cdot d_1^j \cdot d_2^k
\end{equation}

\subsection{LinearPlusForest}

We use a simple additive architecture:
\begin{equation}
    \text{output} = W_{\text{base}} \mathbf{x} + \mathbf{b} + \gamma \cdot \text{Forest}(\mathbf{x})
\end{equation}
where $\gamma$ is a learned shrinkage factor (initialized to 0.1). The linear base preserves residual stream structure; the forest adds input-adaptive nonlinear correction.

\subsection{Unfused QKV Routing}

Q, K, V serve fundamentally different functions. We use \textbf{separate forests} for each, allowing independent routing specialization. Shared routing forces identical paths for all three, limiting expressiveness.

\subsection{QK-Norm}

Tree projections have unpredictable output scale during training. Following Gemma~2/ViT-22B, we apply LayerNorm to Q and K after projection, stabilizing attention logit magnitudes.

\subsection{Speed Optimizations}

We explore a progression of techniques:
\begin{enumerate}
    \item \textbf{Oblivious trees}: Fewer parameters and vectorized leaf prob computation.
    \item \textbf{Alternating layers}: Only every other block uses tree projections.
    \item \textbf{Selective V+O}: Trees only on V and O projections; Q/K use linear.
    \item \textbf{\texttt{torch.compile}}: Fuses operations and optimizes the graph.
\end{enumerate}

\subsection{MoE Variant}

As an ablation, we replace tree forests with standard MoE layers using top-k expert routing, testing whether the tree structure itself matters or any input-dependent projection mixing suffices.

\subsection{Training Details}

\textbf{Per-node temperature:} Each node learns its own temperature via $\text{softplus}(\text{logit} + 0.5413)$ (init.\ 1.0).

\textbf{Temperature annealing:} Held at 1.0 for 50\% of training, then cosine-annealed 1.0 $\to$ 0.7. The 0.7 floor preserves soft routing advantages.

\textbf{Initialization:} $\mathcal{N}(0, 0.1)$ for decision weights (sigmoid range [0.31, 0.69] at $d_{\text{model}}=64$).

\textbf{Optimizer:} Decision weights get $3\times$ LR, zero weight decay. 100-step warmup + cosine decay to 10\%.

\textbf{Regularization:} Entropy reg ($\lambda\!=\!0.005$, depth-decay $2^{-d}$). Leaf balancing ($\alpha\!=\!0.01$).

\section{Analysis and Discussion}

\subsection{Why LinearPlusForest Works}

\begin{enumerate}
    \item \textbf{Linear base provides a guaranteed gradient path.} Even if tree routing shifts unpredictably, the linear projection keeps learning.
    \item \textbf{Trees as corrections, not replacements.} The forest learns the residual between what a linear projection can represent and the optimal projection.
    \item \textbf{Input-adaptive projections.} The effective projection is $W_{\text{base}} + \gamma \cdot W_{\text{eff}}(\mathbf{x})$, a form of input-conditional computation.
\end{enumerate}

\subsection{Why Unfused Routing Matters}

Q, K, V, and O serve fundamentally different functions. Fused QKV routing forces identical tree paths for all three, limiting expressiveness. Unfused routing allows each projection to specialize its feature partitioning.

\subsection{Oblivious Trees and Speed/Accuracy Trade-off}

Oblivious trees achieve comparable accuracy with fewer parameters and faster computation. The shared-hyperplane constraint acts as a regularizer. Alternating layers are effective because not every transformer block benefits equally from nonlinear projections.

\subsection{Limitations}

\begin{enumerate}
    \item \textbf{Speed:} Even the fastest tree variant is slower than standard attention. GPU benchmarks needed.
    \item \textbf{Scale:} Only tested at small model sizes. Scaling behavior unknown.
    \item \textbf{Task diversity:} Only character-level Shakespeare evaluated.
    \item \textbf{Temperature:} The schedule is hand-tuned; adaptive methods could improve robustness.
\end{enumerate}

\section{Related Work}

\textbf{Soft Decision Trees.} \citet{irsoy2012soft} introduced soft trees. \citet{frosst2017distilling} used them for distillation. \citet{hazimeh2020tree} proposed differentiable trees for tabular data.

\textbf{NODE.} \citet{popov2020neural} introduced Neural Oblivious Decision Ensembles for tabular data, using oblivious trees with entmax. We adapt oblivious trees for attention projections with sigmoid routing.

\textbf{Tree-based Neural Networks.} Deep Neural Decision Forests \citep{kontschieder2015deep} combined forests with neural features. Adaptive Neural Trees \citep{tanno2019adaptive} learned tree structure alongside parameters.

\textbf{Mixture of Experts.} LinearPlusForest shares conceptual similarity with MoE \citep{shazeer2017outrageously}. We directly compare against an MoE variant.

\textbf{Efficient Attention.} Low-rank \citep{wang2020linformer}, sparse \citep{child2019generating}, and kernel-based \citep{katharopoulos2020transformers} methods. Tree-based projections offer a distinct inductive bias---input-conditional piecewise linear projections.

\section{Future Work}

\begin{enumerate}
    \item \textbf{GPU optimization:} Custom CUDA kernels for tree routing could reduce the speed gap.
    \item \textbf{Larger scale:} Evaluate on larger models and datasets.
    \item \textbf{Adaptive temperature:} Replace fixed schedule with learned temperature.
    \item \textbf{Routing analysis:} Investigate what features routing learns to partition on.
    \item \textbf{Hybrid architectures:} Trees in FFN gating, embeddings, etc.
\end{enumerate}

% ══════════════════════════════════════════════════════════════════════════
\section{Conclusion}

We demonstrate that differentiable decision trees can serve as effective projection layers in transformer attention, matching standard linear projections on character-level Shakespeare (38.3\% vs.\ 38.7\% validation accuracy). The key insight is that trees work best as \textit{residual corrections to a linear base} with \textit{unfused per-projection routing}.

Speed-optimized variants using oblivious trees, alternating layers, and selective V+O placement achieve 36.4\% accuracy at 171ms/step---significantly faster than full Linear+Forest (545ms/step). The deeper insight is that soft routing is not a compromise---it \textit{is} the feature, computing input-adaptive projections.
